# 以太网→CAN/UDS MCU固件更新系统

## 📋 项目概述

这是一个完整的嵌入式固件更新解决方案，包括：
- **PC上位机** (PyCharm开发) - 通过以太网与主MCU通信
- **主MCU端** (STM32 + W5500) - 作为网关，通过CAN/UDS向从MCU下发固件
- **从MCU** (Bootloader) - 接收CAN/UDS命令，更新固件并自动重启
- **��定义协议** - 支持文件信息、分块传输、校验等

## 🚀 快速开始

### PC端 (Python)

#### 安装依赖
```bash
pip install -r requirements.txt
```

#### 运行GUI
```bash
python gui_updater.py
```

#### 使用步骤
1. 输入主MCU的IP地址和端口 (默认 192.168.1.100:8888)
2. 点击 "连接" 建立以太网连接
3. 在 "新增控制器 ID" 输入框中输入控制器ID (0-255)
4. 点击 "添加" 添加控制器
5. 双击表格中的控制器行，选择对应的Hex或BIN文件
6. 点击 "查询在线状态" 检查控制器是否在线
7. 点击 "开始升级所有" 开始固件更新
8. 实时查看日志输出，监控升级进度

### 主MCU端 (C/STM32)

#### 硬件要求
- STM32F4 或更高版本 MCU
- W5500 以太网模块 (或内置以太网MAC)
- CAN收发器 (如 TJA1050)
- 以太网 RJ45 连接器

#### 编译烧录
```bash
# 使用Keil或STM32CubeIDE编译
# 将生成的固件烧录到主MCU
```

#### 网络配置
- 配置W5500的IP地址、网关、子网掩码
- 确保与PC在同一网段
- CAN总线速率: 500kbps (可配置)

### 从MCU端 (Bootloader)

#### 可选实现 (参考代码)
- 监听特定CAN ID接收UDS服务
- 解析Intel HEX格式数据
- 写入Flash存储区
- 完成后验证和重启

## 📡 协议设计

### 帧格式

```
╔════════════════════════════════════════════╗
║            协议帧结构 (16+ 字节)          ║
╠════════════════════════════════════════════╣
║  魔数 "FWUP"  │  4字节                    ║
║  帧类型        │  1字节  (0x01-0x05)      ║
║  控制器ID      │  1字节  (0=主MCU)        ║
║  序列号        │  2字节  (大端)           ║
║  载荷长度      │  4字节  (大端)           ║
║  载荷数据      │  可变   (0-4096字节)     ║
║  CRC-32校验    │  4字节  (大端)           ║
╚════════════════════════════════════════════╝
```

### 帧类型

| 类型 | 值  | 说明 | 载荷 |
|------|-----|------|------|
| HANDSHAKE | 0x01 | 握手 | 版本字符串 |
| FILE_INFO | 0x02 | 文件信息 | 文件类型、地址、大小、名称 |
| DATA_BLOCK | 0x03 | 数据块 | 块索引、大小、数据 |
| CTRL_INFO | 0x04 | 控制器信息 | 控制器ID、操作码 |
| STATUS | 0x05 | 状态反馈 | 控制器ID、状态、进度、错误码 |
| ERROR | 0xFF | 错误 | 错误码 |

### 通信流程

```
PC                          主MCU                    从MCU
│                            │                        │
├─────────握手──────────────→│                        │
│                            │                        │
├─────────文件信息──────────→│                        │
│                            │                        │
├─────────数据块0───────────→│────→UDS服务0x36────→  │
│                            │                    (CAN)│
│                            │                        │
├─────────数据块1───────────→│────→UDS服务0x36────→  │
│                            │                    (CAN)│
│                            │                        │
├──────────...───────────────→│─────────...──────────→│
│                            │                        │
├─────────升级完成──────────→│────→UDS服务0x22────→  │
│                            │                    (CAN)
│                            │                    │ (写Flash)
│                            │                    │ (校验)
│                            │◄────────CAN回复────┤
│◄─────────状态确认─────────┤                    │
│                            │                  (重启)
│                            │                        │
```

## 🔍 故障排除

### 连接失败
- 检查主MCU IP地址和端口是否正确
- 检查防火墙是否阻止TCP连接
- 检查网络连接和以太网模块
- 尝试ping主MCU IP地址

### 控制器离线
- 检查CAN总线连接
- 检查控制器ID是否正确 (0-255)
- 检查从MCU是否上电
- 查看CAN总线示波器波形

### 升级失败
- 检查Hex/BIN文件格式是否正确
- 检查起始地址和文件大小设置
- 查看详细日志输出中的错误信息
- 尝试重新连接并重新升级
