# 协议设计文档

## 1. 协议概述

### 1.1 设计目标

- **可靠性**: CRC-32校验，超时重传机制
- **灵活性**: 支持多种控制器，自定义负载
- **高效性**: 最小帧头开销，可配置块大小
- **可扩展性**: 易于添加新的帧类型和服务

### 1.2 通信模型

- **拓扑**: 星型 (PC ↔ 主MCU ↔ 从MCU)
- **通道**: PC ↔ 主MCU 使用以太网TCP/IP
- **通道**: 主MCU ↔ 从MCU 使用CAN/UDS
- **工作模式**: 半双工 (请求-响应)

## 2. 帧格式

### 2.1 完整帧结构

```
字节位置      内容              字节数    字节序
──────────────────────────────────────────────
0-3           魔数 "FWUP"      4        ASCII
4             帧类型            1        大端
5             控制器ID          1        大端
6-7           序列号            2        大端
8-11          载荷长度          4        大端
12-11+PL      载荷数据          可变     
12+PL-15+PL   CRC-32            4        大端
```

其中 PL = 载荷长度 (Payload Length)

### 2.2 字段说明

#### 魔数 (Magic Number)
- 固定值: "FWUP" (46 57 55 50 hex)
- 用途: 帧同步和验证
- 长度: 4字节

#### 帧类型 (Frame Type)
- 0x01: HANDSHAKE - 握手
- 0x02: FILE_INFO - 文件信息
- 0x03: DATA_BLOCK - 数据块
- 0x04: CTRL_INFO - 控制器信息
- 0x05: STATUS - 状态反馈
- 0xFF: ERROR - 错误

#### 控制器ID (Controller ID)
- 0: 主MCU (reserved)
- 1-255: 从MCU地址
- 对于多个从MCU，每个都有唯一ID

#### 序列号 (Sequence Number)
- 16位递增计数器
- 用于请求/响应匹配
- 范围: 0x0000 - 0xFFFF

#### 载荷长度 (Payload Length)
- 32位无符号整数
- 表示载荷数据的字节数
- 最大: 4GB (实际受MTU限制)

#### CRC-32校验 (Checksum)
- 32位无符号整数
- 计算: 对帧头(12字节) + 载荷数据
- 多项式: 0xEDB88320 (标准CRC-32)
- 初值: 0xFFFFFFFF, 最终异或: 0xFFFFFFFF

## 3. 载荷格式

### 3.1 HANDSHAKE (0x01) 握手

**请求**:
```
字节 0-N    版本字符串 (ASCII)
```

示例: "PC-Updater/1.0"

**响应**:
- 帧类型: STATUS (0x05)
- 载荷: StatusPayload (见下文)

### 3.2 FILE_INFO (0x02) 文件信息

```
Offset  字段                    字节数    类型
──────────────────────────────────────────
0       文件类型               1        uint8
        - 0: Intel HEX
        - 1: 二进制 (BIN)
        - 2: ELF (预留)
1       目标地址类型           1        uint8
        - 0: Flash
        - 1: SRAM
        - 2: EEPROM
2-5     起始地址               4        uint32 (大端)
6-9     文件大小               4        uint32 (大端)
10-41   文件名称               32       ASCII (null-padded)
────────────────────────────────────────
总长度: 42字节
```

### 3.3 DATA_BLOCK (0x03) 数据块

```
Offset  字段                    字节数    类型
──────────────────────────────────────────
0-1     块索引                 2        uint16 (大端)
2-3     块大小                 2        uint16 (大端)
4-N     块数据                 可变      uint8[]
──────────────────────────────────────────
总长度: 4 + 块大小
```

**特性**:
- 块索引从0开始
- 块大小可变，最大256字节
- 最后一块可能小于256字节

### 3.4 CTRL_INFO (0x04) 控制器信息

```
Offset  字段                    字节数    类型
──────────────────────────────────────────
0       控制器ID               1        uint8
1       操作码                 1        uint8
        - 0: 查询 (Query)
        - 1: 准备升级 (Prepare)
        - 2: 开始升级 (Start)
        - 3: 完成升级 (Complete)
2-N     数据 (可选)            可变      
──────────────────────────────────────────
最小长度: 2字节
```

### 3.5 STATUS (0x05) 状态反馈

```
Offset  字段                    字节数    类型
──────────────────────────────────────────
0       控制器ID               1        uint8
1       状态码                 1        uint8
        - 0: 空闲 (Idle)
        - 1: 忙碌 (Busy)
        - 2: 成功 (Success)
        - 3: 错误 (Error)
2       进度百分比             1        uint8 (0-100)
3       错误码                 1        uint8
        - 0: 成功
        - 1: 无效帧
        - 2: 文件未找到
        - 3: 校验失败
        - 4: 写入失败
        - 5: 超时
        - 6: 未知控制器
──────────────────────────────────────────
固定长度: 4字节
```

## 4. 通信流程

### 4.1 升级流程

```
┌─ PC ─────────────────┬─ 主MCU ─────────────┬─ 从MCU ──────┐
│                      │                    │              │
│  1. 握手             │                    │              │
├─────握手帧──────────→│  (接收)            │              │
│                      │  2. 解析握手       │              │
│                      │  3. 发送STATUS    │              │
│◄─────STATUS────────┤                    │              │
│                      │                    │              │
│  4. 发送文件信息    │                    │              │
├─────FILE_INFO──────→│  (接收)            │              │
│                      │  5. 解析文件信息  │              │
│                      │  6. 准备缓冲区    │              │
│                      │  7. 发送READY    │              │
│◄─────STATUS────────┤                    │              │
│                      │                    │              │
│  8. 发送数据块0    │                    │              │
├────DATA_BLOCK(0)───→│  (接收/缓存)      │              │
│                      │  9. 发送ACK      │              │
│◄────STATUS─────────┤                    │              │
│                      │                    │              │
│ 10. 发送数据块1    │                    │              │
├────DATA_BLOCK(1)───→│  (接收/缓存)      │              │
│                      │                    │              │
│         ...         │       ...         │      ...    │
│                      │                    │              │
│ 11. 发送完成信号   │                    │              │
├─────CTRL_INFO(3)───→│ 12. 解析载荷      │              │
│                      │ 13. 通过CAN/UDS发送给从MCU    │
│                      ├───UDS服务──────→│ 14. 接收数据│
│                      │                  │ 15. 写Flash│
│                      │                  │ 16. 验证   │
│                      │◄───CAN应答────┤ 17. 重启  │
│                      │ 18. 发送SUCCESS│              │
│◄─────STATUS────────┤                    │              │
│                      │                    │              │
└──────────────────────┴────────────────────┴──────────────┘
```

### 4.2 查询流程

```
PC → 主MCU: CTRL_INFO (operation=0)
主MCU → 从MCU: CAN查询
从MCU → 主MCU: CAN回复
主MCU → PC: STATUS (error_code=0 表示在线)
```

## 5. 超时和重传

### 5.1 超时设置

| 操作 | 超时时间 | 说明 |
|------|---------|------|
| 握手 | 2秒 | 初始连接 |
| 文件信息 | 2秒 | 主MCU处理文件头 |
| 数据块 | 2秒 | 普通数据块传输 |
| 完成信号 | 3秒 | 允许从MCU更长的处理时间 |
| 查询 | 2秒 | 控制器在线状态检查 |

### 5.2 重传策略

- **自动重试**: 3次
- **指数退避**: 不使用 (固定间隔)
- **重试间隔**: 50ms
- **放弃条件**: 3次失败后报错

## 6. CRC-32计算

### 6.1 算法

```python
def calc_crc32(data: bytes) -> int:
    crc = 0xFFFFFFFF
    for byte in data:
        crc ^= byte
        for _ in range(8):
            crc = (crc >> 1) ^ (0xEDB88320 if (crc & 1) else 0)
    return crc ^ 0xFFFFFFFF
```

### 6.2 示例

```
输入: "FWUP" + 帧头(8字节) + 载荷(N字节)
输出: 4字节CRC-32值 (大端)
```

## 7. 安全考虑

### 7.1 当前保护

- ✓ CRC-32校验 (检测传输错误)
- ✓ 魔数检查 (防止误识别)
- ✓ 序列号验证 (防止重放)

## 8. 性能指标

### 8.1 吞吐量

- 理论最大: 8 Mbps (以太网TCP/IP)
- 实际 (256字节块): ~2-4 Mbps
- CAN总线: 500 kbps (可配置)
- 升级1MB固件: ~5-10秒 (以太网)

### 8.2 延迟

- 单块传输: ~100ms (含ACK)
- 握手: ~50ms
- 往返时间 (RTT): ~20-50ms
